# PT Feeder 1.7.0 Beta 3

# Added

# Dynamic trend windows (Mandatory change)

WARNING

This is a mandatory change you will need to make.

It is now possible to add as many time trends as you want. In the PtFeeder section of your appsettings, you NEED to replace all MinutesToMeasure trend with

"MeasureTimes": [ 120, 360, 720 ],

All other sections have backward compatibility.

In custom conditions, you can then use for example:

[pair.PercentageChange(3)] > [config.PriceChangeFor12h]

If you use the Medium/Long in your custom conditions, these will need to be changed:

For example:

There is no MediumVwapChange or LongTopCoinsChange on the market object. There is VwapChange(1), VwapChange(2) or TopCoinsChange(3)

# Previous values available

You can now access the previous values for use in custom conditions to measure change. They are nested up to the 3 previous values.

[pair.PercentageChange] - [pair.Previous.PercentageChange] > [config.ChangeInPricePercentage]

# Ability to ignore offsets

A property can be added on a grouping to ignore offsets in that group.

 "PriceTrendChangeGrouping": {
   "IgnoreOffsets": "true",
    "Configs": [
      {
        "MaxPriceTrendPercentageChange": "-10"
      },

# Ability to Enable or Disable a grouping for coins

Properties can be added to enable to disable a grouping for certain pairs. By default, all pairs are included:

"WikiExampleGrouping": {
    "Condition": "[market.BaseCoinPriceChange] - [pair.PercentageChange] <= [config.TriggerName] ",
    "EnabledMarkets":   "ETH,ETC,MTL",
    "DisabledMarkets":  "BCC,XRP",
    "Configs": [
      {
        "TriggerName": "-1",
        "Override": {
          "DcaRebuyTimeout": "30",
          "Weight": "100"
        }
      },
    ]
  },

# Add ability to offset Min/Max trailing buy and profit

Now the boundaries for the calculated trailing buy and profit can be offset like other properties.

# Change the default candle size that PTF uses

By default, PT Feeder uses 5 min candles. This can be changed by adding the property FeederCandleSize in the PtFeeder section of your appsettings. Possible values are: 1m, 5m, 30m, 1h, 1d. Note to make sure your MeasureTimes correspond or you will get an error. For example, a measure time of 120 mins will NOT work with 1 day candles.

# Changed

# Reduced pairs written to ProfitTrailer config

Reduce the number of pairs written to your ProfitTrailer config. Only coins which you hold a position in and are above your MinBuyVolume and MinBuyPrice will be written to the config.

# Reduce the comments

Change the comments on the PT config file to be on one line for different time trends.

# Other fixes

  • Fix BB Std always rounded up
  • Fix PendingOrderWaitTime converted to decimal value

# Removed

# SellOnlyModeEnabled taking precedence

WARNING

This is a mandatory change you will need to check.

Removed SellOnlyModeEnabled taking precedence. This is prior to having weights and overrides. If you have anything like this in your appsettings:

For example:

 "NewCoinsGrouping": {
    "Configs": [
      {
        "CoinAge": "3",
        "SellOnlyModeEnabled": "true",
        "TimeUnit": "Days"
      }
    ]
  },

Please change it to:

  "NewCoinsGrouping": {
    "Configs": [
      {
        "CoinAge": "3",
        "TimeUnit": "Days",
        "Override": {
          "SellOnlyModeEnabled": "true",
          "Weight":  "300" 
        } 
      }
    ]
  },

# PT Feeder 1.7.0 Beta 2

# Added

# Volume Weighted Moving Average

  • Add VWMA as a property on the pair which compares the VWMA over two periods. These two properties need to be added into yhe PtFeeder section of your appsettings to use it. It can be used in custom conditions with the property pair.VwmaPercentageChange
 "VwmaLength": "24",
 "VwmaLookBackPeriod": "34",

Note Squiggles will be releasing a TV script soon

# Long and short EMA and SMA on base coin

  • Add long and short EMA and SMA calculations on the Base coin to allow market conditions to be set based on these values. Add these to your PtFeeder section of your appsettings. Properties are market.BaseCoinShortSma, market.BaseCoinLongSma, market.BaseCoinShortEma and market.BaseCoinLongEma.
  "EmaShortCandleSize": "1d",
  "EmaLongCandleSize": "1d",
  "SmaShortCandleSize": "1d",
  "SmaLongCandleSize": "1d",

  "EmaShortPeriod": "5",
  "EmaLongPeriod": "20",
  "SmaShortPeriod": "5",
  "SmaLongPeriod": "20",

Possible candle sizes are 1m, 5m, 30m, 1h or 1d.

As an example,

"MarketConditionsGrouping": {
    "Condition": "[market.BaseCoinShortSma] - [market.BaseCoinLongSma] < [config.SmaDifference]",
    "Configs": [
      {
        "Name": "Bear",
        "SmaDifference": "0"

      },
      {
        "Name": "Bull",
        "SmaDifference": "100000"
      }
    ]
  },

# Average True Range Percentage of top coins and base coin

  • Add ATRP available for market condition checks to change market conditions based on volatility
market.BaseCoinAtrp
market.MediumTermBaseCoinAtrp
market.LongTermBaseCoinAtrp

market.TopCoinsAtrpAverage
market.MediumTermTopCoinsAtrpAverage
market.LongTermTopCoinsAtrpAverage

# Pair specific VWAP and ATRP

  • Add VWAP and ATRP as a pair specific property for use in custom market conditions
pair.Atrp
pair.AtrpMediumTerm
pair.AtrpLongerTerm

pair.Vwap
pair.VwapMediumTerm
pair.VwapLongerTerm

# PT Feeder 1.7.0 Beta 1

# Added

# PT DCA log and Pairs log available in PT Feeder (Mandatory change)

WARNING

This is a mandatory change you will need to make.

  • Create custom conditions based on ProfitTrailer positions. ProfitTrailerServerApiToken1 needs to be added to hostsettings. See ProfitTrailer wiki on how to set this here. The object positions is available and it has the following properties:
positions.TotalDcaCurrentValue
positions.TotalDcaCurrentValue
positions.TotalPairsBoughtCost
positions.TotalDcaBoughtCost
positions.PairsBalance
positions.DcaBalance
positions.RealBalance
positions.DcaCoinCount
positions.PairsCoinCount

# Choose your top coins for your top coin trend

  • Choose which coins you would like your TopCoinTrendChange to be built off. Add TopCoinTrendChangeCoins to your PtFeeder section of appsettings to use this feature.

# Comments on which configs from appsettings were used

  • Comments on pairs/dca config about which conditions were used from your appsettings.

# Create a custom market condition grouping

  • Add ability to create a custom MarketCondition grouping config so something like this is possible github issue (45):
Condition: "[market.BaseCoinPriceChange] - [market.TopCoinsChange] < [config.SomePropertyFromConfig]"

# Override and calculate each trailing individually

  • Added DcaMaxTrailingBuy/DcaMinTrailingBuy and DcaMaxTrailingProfit/DcaMinTrailingProfit in PT Feeder section.

If a user has TrailingBuy/TrailingProfit and/or DcaTrailingProfit/DcaTrailingBuy in their Defaults section, we assume that they DO NOT want to calculate trailings. They can use a combination this way, so they can have a fixed DcaTrailingProfit but calculate DcaTrailingBuy for example.

  • Added discord messages on startup
